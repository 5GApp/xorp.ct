XORP Live CD

The XORP Live CD is a bootable CD containing the Linux operating system and the XORP routing code. It runs on an x86 PC only.

Linux boots off the CD into a memory filesystem, and then starts up XORP directly. If there is a DOS floppy in the floppy drive, the XORP configuration file will be taken from the floppy and used for the router configuration. If the PC has a hard disk, that disk will not be modified by running the Live CD.

The Live CD serves a number of purposes:

  • It's an easy way to try out XORP without needing to compile anything or reformat the disk on your PC.
  • It's a quick way to get a relatively secure router on demand - you don't need to know anything about Linux.
  • It's a great tool for a student lab session, requiring no installation.

Getting the XORP Live CD

The Live CD is not yet available. It will be available from this page when the XORP 1.0 release is made.

The XORP Live CD will be available as an ISO image. You need to download it and burn it onto a CD.

Running the XORP Live CD

To run the Live CD will probably require you to modify your PC's BIOS settings. In particular, the boot order should be:
  1. CD drive.
  2. Floppy Disk.
  3. Hard Disk.
Actually it doesn't matter what order the floppy and hard disk are, so long as they're after the CD drive in the boot order. This is usually pretty easy to change in the BIOS - you might want to make a note of the original boot order in case you want to switch it back afterwards.

If you want the router to start up with a non-default configuration, you'll also need to insert a DOS-formatted floppy into the floppy drive, and put the XORP config.boot file you want used on it. See "Getting Started" for information about what goes in config.boot.

Then reboot the PC. The PC should boot from the CD, start the XORP router manager, read the config.boot file from the floppy, and configure the interfaces and routing protocols as specified in the config.boot file.

If there's no config.boot file on the floppy, then a default configuration will be loaded with no routing protocols running, and the interfaces configured with private addresses.

The default user is xorp with password xorpxorp. Such a machine should not be connected to the public Internet without changing the default password. If you want to connect the machine and use it for routing, you will also need to include a passwd file on the floppy. Details of how to create this passwd file will appear here when the LiveCD is released.
