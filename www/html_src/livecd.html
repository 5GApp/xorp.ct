  2. Boot it up.
  3. If you want to use XORP all by itself, it is installed in /usr/local/xorp. You can create a config file and start it manually: xorp_rtrmgr -b [cfg-file]

Using LANforge and Netsmith to create XORP routers.

LANforge is a network testing tool produced by Candela Technologies. It can be used for network traffic generation and network emulation (including latency, packet loss, bandwidth-constriction, etc). As part of its virtual-routing feature set, it can manage XORP through its Netsmith GUI widget. As long as you do not attempt to generate traffic or add network impairment, you do not need any LANforge licenses.

  1. After booting up the Live-CD, start LANforge and the LANforge-GUI by clicking on the icons on the desktop.
  3. Click 'Netsmith' in the Status panel of the LANforge-GUI. It will be towards the middle of the screen.
  4. In Netsmith, right-click to create a new Virtual Router and then drag network interfaces into the router to use them. The Virtual Router has different routing options, including OSPF, BGP, etc.
  5. You may need to configure the Interfaces as well..right-click on the little green or black squares in Netsmith to modify them.
  6. You may optionally create additional virtual routers and connections between the virtual routers. When creating connections, be sure to skip the 'B' ports and the WanLink, otherwise you will need LANforge-ICE emulation licenses for the WanLink to function.
  7. When satisfied, click Apply and XORP will be (re)started with the proper config.
  8. The xorp config files are saved in /home/lanforge/vr_conf/ and may be useful as a starting point for more advanced XORP configurations.
  9. You may wish to see some of the LANforge cookbook examples for more details. These examples are tailored to using LANforge, but the ones related to routing will be using XORP.
